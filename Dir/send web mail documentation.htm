The Send Web Mail process allows sending an email message over the web using any web browser.  Since the messages are HTML format, tags, such as bold (<b></b>) and underline (<u></u>), can be used to enhance the message text.  Attachments can also be sent, but the file must be located on the server not the workstation.

 

Name / Email

Name and Email address of the sender.

Subject

Message title

TO, CC, BCC

Email address of each recipient.  Separate multiple entries with a semicolon.

The BCC prompt will only appear when a BCC address has been selected using the Directory function.

Send

When pressed, the message will be sent to the addresses entered into the TO, CC and BCC address areas, unless the “Shortcut” option was checked.

Directory

Invokes the Email Address Directory process to lookup and select the TO, CC and BCC email addresses.  To bypass the directory search criteria window, enter “?CCC” in the SUBJECT, TO, CC, BCC or MESSAGE area, where “CCC” is the criteria to use for the directory search.  For example, enter “?Fulcrum AND John” to retrieve all the entries that contain “Fulcrum” and “John” in any of the directory fields.

Preview

Display the formatted version of the message.  From the Preview window click on the Back button to return to the email composition window.  Do not use the browser’s back function or the Send Web Mail prompts may not get properly refreshed.

Templates

List all of the available template files for selection.  The appropriate fields will be updated from the parameters in the selected template file. 

Attach Files

This option allows attaching files to the current email for sending.  Because the program is web-based, the files to be attached must be located in the designated web site mail file storage folder, making this a two-step process.

 

·         When the option is invoked, the “Upload Files” page will appear.  From this page the files to be attached are transferred from the workstation to the web site email storage folder.  This target folder must have both write access to allow the upload and read access to allow attaching files to an email message.

·         Once the files have been uploaded, the email screen will re-appear with the attachment input prompt pre-filled with the uploaded file names. 

 

If no files are uploaded, the email program will issue a prompt to capture the file specifications used to build the file attachment selection window.  The file specification must contain the web site folder where the files are located, prefixed with a backslash, and the optional file specification prefixed with an “*”.  For example, entering \DOCS\ would list every file in the DOCS folder.  Entering \DOCS\*.DOC would list only those files in the DOCS folder that contain “.DOC”.  Use multiple asterisks for multiple specifications.  For example, \DOCS\*CLIENT*.DOC” would display all files that contain both “CLIENT” and “.DOC”.

 

Once files have been attached, the Attachment prompt will appear with each attached file name followed by the name of the file the recipient receives.  Initially both of these will be the same file name.  If you want the recipient to receive the file with a different name, change the value within the parentheses.

Clear

This option will clear all of the fields on the window except the Sender and To prompts.

  

  

Other Options

 

The following options are also available when sending an email message:

 

·     BCC Sender:  Mail a blind carbon copy of the message to the sender’s email address for storing in the Send folder.

·     Return Receipt: Request notification when a sent message has been opened.

·     Text Format: Send the email in text format instead of the default HTML format.

·     Shortcut:  Build a link to this process that can be copied and pasted into a shortcut.  See below for additional details.

 

 

Template Files

 

Template files can be used to pre-fill email recipients, sender details, message, attachments as well as several options.  These files must be in the default server file folder and have the extension “EM”:  The following optional parameters are used to define the information stored in the template:

 

·              SN:XXX where XXX is the sender’s name

·              SE:XXX where XXX is the sender’s email address

·              SB:XXX where XXX would be the message subject

·              TO:XXX, CC:XXX, BCC:XXX where XXX is the recipient’s email addresses.  Multiple entries must be separated by a semi-colon

·              BCS:Y turns on the “bcc Sender” option

·              TF:Y turns on the “Text Format” option

·              RR:Y turns on the “Return Receipt” option

·              AF:XXX where XXX is the name of a file attachment or lookup specification. 

·              Comment Lines have a colon in the first position and will be ignored.

·              If a line is not identified as one of the above parameters, it is assume to be part of the message text. 

 

To select an existing template, click on the Templates button, which will display a list of all the available template files for selection.  The template option can also be invoked from a shortcut by including the “FN=<XXX>.EM”, where “XXX” is the name of the template file without the extension.

 

Template files can be very useful.  For example, you could have a template file setup with only the SN: and SE: options for every email address you use.  Selecting the appropriate template will populate the sender details for the appropriate email account.

 

 

HTML Customization

 

A special configuration file, <logon username>_EMHTML.CFG, can be used to customize what the HTML message will look like to the recipients.  The file will contain the following details:

 

·         Lines that begin with a colon are considered comments and will not be processed.

·         The first line that begins with an HTML tag will be used to surround the message when mailed. For example, if the HTML string is “<body bgcolor=”blue”><font face=’arial’></font>”, the message would begin with the body and font tags and would end with “</font>” tags.  Do not include the <html>, </html> or closing </body> tags.  These will be added when the message is mailed.

·         The remaining lines are considered signature lines attached to the end of the message when mailed.  The first part of each signature line is either the email address the signature is assigned too or the word “ALL” for assignment to all messages.

 

If any of the following variables are imbedded in a template file message or within the signature file text, the variable will be replaced with the appropriate values.  The variables are case sensitive and must be enclosed in brackets.

·         [ToFName] – First name of the recipient

·         [ToLName] – Last name of the recipient

·         [SenderName] – Sender’s full name

·         [SenderEmail] – Sender’s email address 

 

For example, for the signature line “ALL <b>[SenderName]</b><br>[SenderEmail]”, the message text will be the sender’s name in bold font with the sender’s email address under the name  in regular font.  Because it starts with “ALL”, this signature will appear at the end of every message regardless of the email address used.  Prefixing an email address with “-“ will suppress the signature function for that particular email address, either in the configuration file or in the senders email prompt. 

 

The HTML tags in _EMHTML.CFG will not be applied if the message body already contains an <HTML tag.

 

 

Email Address Directory Customization

 

If the Email Address Directory customization option is invoked, the Send Web Mail sender function can be customized using the file <username>_DIREMDEFAULTS.CFG.  Each line in the this file will contain the following parameters, separated by a comma:

 

·         The first word or phrase used in the Email Address Directory customization search file (i.e. Fulcrum or Case Partners). 

·         Sender’s email address to use when the previous condition is satisfied (i.e. dammons@casepartners.com)

·         Sender’s name (i.e. Dean Ammons)

 

When a match is found, the Send Web Mail sender information will be pre-filled with the appropriate sender information in the _DIREMDEFAULTS.CFG file.   Using the above example, if the directory search was customized to only retrieve “Case Partners” entries, the sender fields would be pre-filled with “Dean Ammons” and dammons@casepartners.com. 

 

 

Composing Messages With MS Word or Workstation Email Program

 

Composing the HTML message with Microsoft Word or an email program such as Outlook Express provides access to a spell checker and a WYSIWYG interface (what-you-see-is-what-you-get).  After composing the message using the desired formatting options (font, color, bold, underline, etc.),

 

  Composing with MS Word

1.       Save the document as Web Page Filtered

2.       Click YES at the filter usage warning prompt

 

  Composing with Email Program

1.       Save the message as an HTML formatted file

 

  Processing HTML file

3.       Open the saved HTML file in notepad.

·         From within Windows Explorer right click on HTML file

·         Select Open With Notepad

4.       Select Edit => Select All

5.       Select Edit => Copy

6.       Paste clipboard contents into the Send Web Mail message area.

 

   

Building Web Mail Shortcut

 

Checking the Shortcut box and clicking on Send will create a link to this process using the currently entered Sender, TO, CC and BCC information.  When this link is invoked, the Send Web Mail window will open and be pre-filled with the saved sender and recipient addresses.   For example, to create a shortcut for the members of a project team;

·     Enter the address for each team member in the “To” prompt,

·     Enter the appropriate information in the Sender prompts, or leave blank,

·     Check the Shortcut option, and

·     Click the Send button.

 

Using the information entered, the process will display the hyperlink text.  Select, copy and paste the text string into the shortcut prompt.  Any of the template options can also be added to the shortcut string.   For example, to use the template option “BCS:Y”, add “&BCS=Y” to the end of the pasted shortcut string.
