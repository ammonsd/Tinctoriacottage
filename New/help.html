Visitor Registry Documentation

Description

The Fulcrum Computing Visitor Registry will allow an Organization to maintain a record of visitors to their Web site using any "guest book" style information collection form. The visitor information collected is stored in a database that can be updated by the Organization.

To remain in contact with visitors to the Web site and keep them informed on news about the Organization, there is a function that builds an e-mail distribution list from the Registry. The distribution list can be narrowed using the member selection function.

Membership Maintenance Menu Options

Add New Members

This option allows adding new members to the Visitor Registry without having to use the Web Site guest book. To facilitate adding multiple members, a cleared form will reappear after each new member is added. Pressing "Submit" without an e-mail address will terminate the "Add" process and return control to the Maintenance Menu.

The input form will have a field called Keywords. This field is provided as an additional option for the member selection function. Keywords can be used to categorize members by geographical regions, products, etc. Multiple keywords must be separated by a comma or space. If the guest book has check boxes, each one checked will appear in the visitors keywords as "CB#", where "#" is the check box number. This allows selecting members based on the boxes checked when the guest book was processed.

Membership Maintenance

After this option is selected, the selection criteria window displays to determine which members to update. Once the criteria has been entered, a selection box will display with the name, e-mail address and keywords of all the members in the Registry that satisfied the requested criteria. From this list, members can be selected for either updating or removal from the Registry. A form, pre-filled with the members current Registry information, will appear for each member selected for updating.

List Members for Printing

This option will retrieve the Registry details for every member that satisfied the requested selection criteria and display the information in a format suitable for printing from the browser.

Send Broadcast e-mail

This option will display a window to collect the details for sending an e-mail message to every member on the Organization's e-mail distribution list that satisfied the requested selection criteria. The "From" address will be the Organization's Customer Service email address and the body of the message will include:

  • Subject: This is the text that will appear in each recipients e-mail message.
  • Text: Either key in the text to be e-mailed or paste in text that was copied to the clipboard. The recommended method would be to compose the message in your e-mail program where it can be formatted, spell checked and saved as a draft copy. The finished contents would be copied and pasted into the text area when ready to send.

An individual e-mail message will be sent to each recipient matching the selection criteria. This process allows:

  • Maintaining a personal touch by showing only the recipients name in the "To" address area,
  • Customizing each e-mail message sent using the parameters outlined below, and
  • Maintain confidentiality by not listing all the recipients on each e-mail message sent.

Customization for reach recipient's e-mail message can be achieve by inserting any of the following tags into the message template. Just prior to sending each message, the tags will be replaced with the appropriate information for that recipient. The tags must be inserted exactly as shown, including lower case and brackets. The preview option will display a sample of what the e-mail text will look like after the substitutions.

  • [company] - Company's Name
  • [custsrvemail] - Customer Service e-mail Address
  • [homepg] - Company's Home Page
  • [mbrfname] - Member's First Name
  • [mbrlname] - Member's Last Name
  • [mbremail] - Member's e-mail Address
  • [mbrcompany] - Member's Company
  • [mbraddr1] - Member's Address #1
  • [mbraddr2] - Member's Address #2
  • [mbrcity] - Member's City
  • [mbrstate] - Member's State
  • [mbrzip] - Member's Zip Code
  • [mbrcountry] - Member's Country
  • [mbrphone#] - Member's Phone Number formatted as (xxx) xxx-xxxx
  • [mbrhomepg] - Member's Home Page

Criteria For Member Selection

The following selection criteria can be entered to narrow the number of members processed for the Maintenance, Printing and Broadcast E-Mail functions. Multiple entries for a category are separated with a comma (i.e. MD,PA,DE for multiple States).

  • First and Last Name - Enter data exactly as entered by member, including upper and lower case and any punctuation.
  • Company - Enter data exactly as entered by member, including upper and lower case and any punctuation.
  • State - Use the two character state code.
  • Zip - Searches the five character portion of zip code only.
  • E-mail Address - Since e-mail addresses are stored in lower case, the criteria entered will be converted to lower case.
  • Keywords - Since keywords are stored in upper case, the criteria entered will be converted to upper case. Matches are on the exact keyword, therefore the wild card option cannot be used. Selection can either be determined by matching a single keyword ("Or" option) or matching every keyword entered ("And" option).
  • Distribution List - Selecting "Yes" will list only members on the Distribution List. Selecting "No" will list only members not on the Distribution List.

Besides the exact match search, the following special options are available:

  • Prefix an entry with "-" to do a not equal type of search (i.e. The State entry -DE will list all members not in Delaware)
  • Use the symbol "*" to indicate a wild card type of search. For example, using the Last Name category:
    • "A*" - Search for all names beginning with "A"
    • "*S" - Search for all names ending with a "S"
    • "*e*" - Search for all names that contain an "e"
    • "-*ith*" - Search for all names that do not contain "ith"
Comparisons default as follows:
  • Multiple entries within a category only require a single match to satisfy the search requirements for that category. The only exception is keywords, were the comparison option is selectable.
      For example, if three states are entered, only one has to match to satisfy the search criteria for the category State.
  • When multiple categories are requested, every category must have at least one match for that member to be selected.
      For example, if the categories State and Last Name are requested, there must be a match of at least one State entry and at least one Last Name entry to satisfy the search criteria.

Inventory and Order Form Documentation

Inventory Maintenance

Inventory items are grouped by the Web page that has the item for sale. The Inventory Maintenance Menu will have an option for each Web page as well as an option to display a combined list of the inventory items for all the Web pages.

Inventory Maintenance is a three step process. Selecting the Web page "Go To" option on the menu will display the Inventory items for that Web page. To review or update inventory, check mark the appropriate items and select the "Process..." option. The details for each item selected will appear in an update form where changes can be made. Leaving the Title blank will delete an item from the inventory list. Changing the quantity of an item to zero will leave the item on the inventory list but the item will not appear on the Web page as an item for sale. Selecting the "Update" option will update the database with the changes. The "Return" option ignores the changes and returns to the selection page.

Selecting the "Add" option will bring up a blank form for entering new inventory items. The blank form will reappear after each item is added. Select the "Return" option to discontinue adding new items and return to the selection page.

The items will appear on the Web page in the sequence displayed on the selection page. The sequence can be changed by changing the "Seq" values and selecting the "Process..." option. If a zero "Seq" is entered, the item will be removed from the inventory list.

Once the changes have been completed, select the "View..." option to review the changes before they are applied to the Web page. The view page will display the items exactly how they will appear on the actual Web page (minus the normal Web page text and graphics). It is important that the view page looks exactly how it should when the Web page is updated and visible by the customers. If not correct, select the "Return..." option to go back to the maintenance page. If the view page is correct, select the "Apply..." option to update the Web page with the inventory items.

Once the changes have been applied, the actual Web page will display with the updated inventory items. At this point the new inventory items are available for your customers.

Order Form

There are two Order Form formats; an interactive format and a printable format. The interactive format displays when a customer has selected the "Order" option on any of the pages built by the Inventory Maintenance function. When the Order Form displays, it will be pre-filled with:

  • Customer information (if previously entered by the Customer),
  • Details of all items selected for purchase (quantity, code, description and price),
  • Sales tax (if applicable)
  • Shipping Cost (calculated based on the Organization's predefined algorithm), and
  • Total order amount.

The customer can modify any of the item quantities and select the "Re-Calculate" option to display the new totals. Entering a zero quantity will remove the item from the Order Form. Items can only be added to the Order Form by selecting it on the Web page where it is listed for sale.

If the "Continue Shopping" option is selected, the Customer will return to the previous Web page where the items were selected. When the Customer returns to the Order Form, any additional items selected will be added to the Order Form with the previously selected items. If the customer decides not to process the Order Form, the selected items will remain on the Order Form for three days.

Note: Any pre-filling of the Order Form, either the Customer information or any selected items, is dependent on the Customer having the browser cookie option activated.

The Printable Format option would be selected once the Customer has finished shopping and is ready to process the order. This function will:

  • Re-display the Order Form in a format that is suitable for printing (no background color, buttons or input fields, etc.).
  • Clear the Customer's selected items cookie,
  • Place the current date and next order number on the form for reference,
  • Send an email to the "Order Department", detailing the Order Form information.

This format would be printed by the Customer and mailed, along with the payment, to the address at the top of the form.
